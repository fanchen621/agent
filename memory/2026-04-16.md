## 自我纠错：Memory Misattribution 教训（21:16 PM）

### 错误事件
在执行"删除狐仙残留"任务时，将柳贯一自己的会话记录（2115行/2157行）错误标记为"狐仙残留"并要求删除。

### 错误链路
grep命中 → 关键词匹配 → 快速归类 → 未验证来源 → 错误标记

### 根因：记忆系统架构缺陷
1. 纯文本日志无来源metadata（source/provenance）
2. grep结果无上下文，纯文本匹配即下结论
3. 缺少"读取确认步骤"（expand + verify before judge）
4. 缺少"来源冲突检测"机制

### 修复：Memory操作强制流程

**所有memory操作必须遵循：**

1. **grep → parse → attach metadata → 再判断**
2. **使用前必须三步**：locate → expand（±20行上下文）→ verify（确认来源）
3. **来源分类**：
   - `system`：系统配置
   - `conversation`：用户对话
   - `agent_self`：自己生成的记录（trust=0.6，不直接用于事实判断）
   - `imported`：外部导入（狐仙记忆等）
4. **结论必须能回溯**：结论 → memory_id → 文件 → 行号 → 上下文
5. **低trust结论**：必须标记 `UNVERIFIED CLAIM`，禁止直接作为事实判断

### 引用链追溯（本次事件）
- 错误结论：2115/2157行是"狐仙残留"
- 实际来源：柳贯一当前会话的自我记录（agent_self）
- 验证：查看原文上下文确认source=agent_self

### 教训
- grep出来的文本必须expand上下文才能判断来源
- agent_self内容≠狐仙内容，不应混淆
- 有疑问时不标记"删除"，先标注"UNVERIFIED待核实"
